# Security Policy

## Supported Versions

MarineShield AI is currently under active development. Security updates will be provided for the latest version of the project.

## Reporting a Vulnerability

If you discover a security vulnerability, please do **not** create a public GitHub issue.

Instead:

1. Contact the project maintainer privately.
2. Include a detailed description of the vulnerability.
3. Provide steps to reproduce the issue if possible.
4. Include the affected files or components.

The maintainer will:

* Acknowledge the report.
* Investigate the issue.
* Release a fix if the vulnerability is confirmed.
* Credit the reporter when appropriate (unless anonymity is requested).

## Security Best Practices

Contributors should:

* Never commit API keys or secrets.
* Store sensitive configuration in environment variables.
* Keep dependencies up to date.
* Validate user inputs.
* Follow secure coding practices.
* Report vulnerabilities responsibly.

Thank you for helping keep MarineShield AI secure.
